Relational Selectors & Combinators

What is a Relational Selector?

Relational selectors target elements based on their relationship to other elements within the markup.

Types of Relational Selectors

Descendant Combinator

E F

The descendant selector targets any element F that is a descendant(child, grandchild, great grandchild, and so on) of an element E

Examples ol li Targets any 'li' element that is a descendant of 'ol'

Child Combinator

E > F

The child combinator matches any element F that is a direct child of element E

It will ignore any elements are further nested.

Examples ol > li This will only target 'li' elements that are directly inside the 'ol' element. Any 'li' elements nested inside a 'ul' element would be ignored.

Next Sibling Selector

E + F

This selects any element F that shares the same parent as E, and comes directly after E in the markup

Examples li + li It will target all li elements except the first li element

Following Sibling Selector

E ~ F

Matches any element F that shares the same parent as any E and comes after it in the markup

Examples h1 ~ h2 Matches any 'h2' that follows any 'h1', as long as they both share the same direct parent
